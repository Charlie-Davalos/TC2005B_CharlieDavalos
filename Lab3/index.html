¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

Permiten comunicar al servidor lo que se quiere hacer

  • GET: Se utiliza para solicitar un recurso específico.
  • HEAD: Es similar a GET, pero solo solicita los encabezados de respuesta.
  • POST: Se utiliza para enviar datos al servidor web como un formulario web o una publicación.
  • PUT: Se utiliza para actualizar o crear un recurso en una ubicación específica en el servidor web.
  • PATCH: Se utiliza para actualizar parcialmente un recurso en lugar de reemplazarlo por completo.
  • DELETE: Se utiliza para eliminar un recurso específico del servidor web.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404?

Sí, el error 404 significa que el servidor web no pudo encontrar el recurso solicitado por el usuario, y esto puede ser causado por una variedad de problemas, como un enlace roto o una página eliminada.
Como desarrollador web, debes ver por la seguridad de tus clientes y arreglar posibles errores.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, el error 500 es un error interno del servidor que indica que se ha producido un error en el servidor al procesar la solicitud del cliente.

¿Cuál es el ciclo de vida de los sistemas de información?

El ciclo de vida de un sistema de información (SLC, por sus siglas en inglés) es un proceso que describe las diferentes etapas por las que pasa un sistema de información, desde la concepción inicial hasta su eliminación.

¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de vida de los sistemas de información (SDLC, por sus siglas en inglés) es un enfoque estructurado para el desarrollo de sistemas de información que consta de una serie de fases interdependientes y secuenciales.
Fases: planificación, análisis, diseño, implementación, pruebas, mantenimiento.
